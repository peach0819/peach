报表逻辑

  1. 默认全局只保留 海拍客 业务. (bu_id = 0) 和 sale_dc_id = -1 
  2. 取消掉原有的 门店类型 的过滤. (该模板新增),通过销售部标识来唯一过滤
  3. 发放对象不为空（该模板新增）
  4. 默认支付时间小于等于方案结束时间
  5. 门店商品的新签时间在方案时间之内

方案汇总指标逻辑

1、如统计指标为新签门店数： 则不重复计数shop_id,  cout(distinct case when 是否新签成功='是‘  and  是否离职=’否‘ then shop_id else null end )

2、如统计指标为新签品牌门店数： 则不重复计数shop_id&brand_id ,  cout(distinct case when是否新签成功='是‘  and 是否离职=’否‘  then shop_id&brand_id else null end )

3、如统计指标为新签GMV： 则求和（实货GMV-退款)    sum( case when 是否新签成功='是‘ and 是否离职=’否‘ then 实货GMV-退款  else 0  end )

4、发放对象不为空

V3需求：

增加大BD新签逻辑不改模板规则的任何逻辑，主要为修改新签中间表的报表逻辑和增加销售部标识的筛选项。

主要改动点如下：

1、新签中间表去掉大客户门店和非服务商订单的过滤条件，选择全部数据 

2、增加销售团队标识字段，并且新的模板规则统一通过销售团队标识过滤，详见上方报表逻辑

3、冻结销售人员不再只取BD，而是根据不同的销售团队标识取不同的值，

如果为BD部，则取岗位为BD的数据，如果为大客户部，则取岗位为大BD的数据（不需要保留id）
