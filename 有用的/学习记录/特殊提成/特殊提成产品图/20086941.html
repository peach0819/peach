一、方案配置前

1、职能不符合当前岗位统计判断

目的：保证 大BD,BD岗位的销售人员对应的 门店上的职能 都是唯一对应的. 

预警规则：职能不符合门店数= 0

预警规则代码：

职能不符合当前岗位  展开源码 
 select
count(distinct shop_id)
from 
(
select
ytdw.get_service_info('service_type:销售',service_info,'service_feature_name') as service_feature_name,
ytdw.get_service_info('service_type:销售',service_info,'service_job_name') as service_job_name,
shop_id,
service_info
from dw_shop_base_d
where dayid='$v_date'
  AND ytdw.get_service_info('service_type:销售',service_info,'service_job_name') in ('大BD','BD','BD新签') 
  AND ytdw.get_service_info('service_type:销售',service_info,'service_feature_name') != ytdw.get_service_info('service_type:销售',service_info,'service_job_name')
)t

2、大BD门店的判断

目的：保证 大BD门店上挂的是大BD,BD门店上挂的是BD

预警规则：不符合门店数= 0

预警规则代码：

大BD门店判断  展开源码 
 select
count(distinct shop_id)
from 
(
select
ytdw.get_service_info('service_type:销售',service_info,'service_feature_name') as service_feature_name,
ytdw.get_service_info('service_type:销售',service_info,'service_job_name') as service_job_name,
shop_id,
service_info
from dw_shop_base_d
where dayid='$v_date'
  AND ytdw.get_service_info('service_type:销售',service_info,'service_job_name') in ('大BD','BD') 
  AND ytdw.get_service_info('service_type:销售',service_info,'service_feature_name') != ytdw.get_service_info('service_type:销售',service_info,'service_job_name')
)t

3、每个销售都有业绩目标

目的：保证目前销售都有业绩目标，且基本目标唯一

预警规则：不符合人数= 0

预警规则代码：

销售业绩目标唯一  展开源码 
select 
  count(*)
from (
  select 
   user_target.assessment_id,
   assessment.month,
   user_target.user_id,
   user_target.target
  
  from(
    select
      assessment_id,
      user_id,
      target 
    from dwd_kpi_indicator_target_d 
    where dayid = '$v_date' and is_deleted = 0
  ) user_target 
  
  left join (
    select 
      id,
      substr(start_time,0,6) as month 
    from dwd_kpi_assessment_d
    -- status 1待生效2生效中3已过期4已失效
    where dayid = '$v_date' and status in(2,3) and is_deleted = 0
  ) assessment on user_target.assessment_id = assessment.id
) tem
where month > '202001'
group by assessment_id,month,user_id
having count(*) >1

 

4、部门变更监控

目的：保证部门变更之后，逆向稽查可以追溯  关于业务规定的部门只能新建，不能修改

预警规则：保证部门id和name 对应关系不变,不符合部门数=0

预警规则代码：

部门id和name对应关系  展开源码 
 select
count(case when id1=id2 and name1!=name2 then id1 else null end) as m
from 
(
select
t1.id as id1,
t2.id as id2,
t1.name as name1,
t2.name as name2
from 
(
select
id,
name
from dwd_department_d
where dayid='$v_date'
) t1 
left join 
(
select 
id, 
name
from dwd_department_d
where dayid=from_unixtime(unix_timestamp(date_add(from_unixtime(unix_timestamp('$v_date','yyyyMMdd'),'yyyy-MM-dd') ,-1),'yyyy-MM-dd'),'yyyyMMdd')
)t2 on t1.id=t2.id
)t

1、关于销售人员的特殊激励

目的：保证销售人员的特殊激励计算合理，防止出现不封顶情况

预警规则：判断销售人员最大的激励是否>=100000,结果不为0则预警

预警规则代码：

销售人员特殊激励  展开源码 
 select
count(1)
from 
(
select
plan_no,
grant_object_user_name,
sum(commission_reward) as commission_reward
from dw_salary_forward_plan_sum_d
where dayid='$v_date'
group by plan_no,
grant_object_user_name
)t
where commission_reward>=100000
