    • 提问一

    现在有一个文件的url，我想要获取这个文件变成一个InputStream，然后如果是.amr类型的，需要转换为.mp3格式，提供一下java代码

    回答一

    以下提供一种基于 JAVE2 库的解决方案，该库封装了 FFmpeg，支持 AMR 转 MP3，且无需系统预装 FFmpeg。
    核心思路：

    1. 从 URL 获取输入流。

    2. 根据 URL 路径后缀判断是否为 .amr 文件。

    3. 若非 AMR，直接返回原始流。

    4. 若为 AMR，先下载至临时文件，再转换为 MP3 临时文件，最后返回封装了自动删除逻辑的 MP3 文件流。

    1. 添加 Maven 依赖


    xml

    <dependency>
    <groupId>ws.schild</groupId>
    <artifactId>jave-all-deps</artifactId>
    <version>3.3.1</version>
</dependency>

    jave-all-deps 包含所有平台的 FFmpeg 二进制，体积较大。若需精简，可替换为 jave-core 配合对应平台的 jave-nativebin-...。

    2. 完整 Java 代码


    java

    import ws.schild.jave.Encoder;
import ws.schild.jave.EncoderException;
import ws.schild.jave.MultimediaObject;
import ws.schild.jave.encode.AudioAttributes;
import ws.schild.jave.encode.EncodingAttributes;

import java.io.*;
import java.net.URL;

/**
 * 从 URL 获取 InputStream，若为 .amr 文件则自动转换为 MP3 流
 */
public class AmrToMp3Converter {

    /**
     * 主方法：根据 URL 返回输入流（非 AMR 直接返回原始流，AMR 返回转换后的 MP3 流）
     *
     * @param urlString 文件 URL
     * @return 输入流（使用后请务必关闭）
     * @throws IOException  网络或文件操作异常
     * @throws EncoderException 音频转换异常
     */
    public InputStream getInputStream(String urlString) throws IOException, EncoderException {
        URL url = new URL(urlString);
        String path = url.getPath();

        // 1. 判断是否为 .amr 文件（忽略大小写）
        boolean isAmr = path != null && path.toLowerCase().endsWith(".amr");

        // 2. 非 AMR：直接返回原始流
        if (!isAmr) {
            return url.openStream();
        }

        // 3. 下载 AMR 文件到临时文件
        File amrTemp = File.createTempFile("audio_", ".amr");
        try (InputStream in = url.openStream(); FileOutputStream fos = new FileOutputStream(amrTemp)) {
            transferTo(in, fos);  // 复制数据
        }

        // 4. 转换为 MP3 临时文件
        File mp3Temp = File.createTempFile("audio_", ".mp3");
        try {
            convertAmrToMp3(amrTemp, mp3Temp);
        } catch (EncoderException e) {
            // 转换失败，清理临时文件并抛出异常
            amrTemp.delete();
            mp3Temp.delete();
            throw e;
        } finally {
            // 转换完成后删除 AMR 临时文件
            amrTemp.delete();
        }

        // 5. 返回 MP3 文件流，并覆写 close() 实现自动删除临时文件
        return new FileInputStream(mp3Temp) {
            @Override
            public void close() throws IOException {
                super.close();
                mp3Temp.delete();   // 流关闭时删除 MP3 临时文件
            }
        };
    }

    /**
     * 使用 JAVE2 将 AMR 文件转换为 MP3
     */
    private void convertAmrToMp3(File source, File target) throws EncoderException {
        MultimediaObject multimediaObject = new MultimediaObject(source);

        AudioAttributes audio = new AudioAttributes();
        audio.setCodec("libmp3lame");   // MP3 编码器
        audio.setBitRate(128000);        // 128 kbps
        audio.setChannels(2);             // 立体声
        audio.setSamplingRate(44100);     // 44.1 kHz

        EncodingAttributes attrs = new EncodingAttributes();
        attrs.setOutputFormat("mp3");
        attrs.setAudioAttributes(audio);

        Encoder encoder = new Encoder();
        encoder.encode(multimediaObject, target, attrs);
    }

    /**
     * 将 InputStream 全部写入 OutputStream（Java 9+ 可用 transferTo，此处提供兼容写法）
     */
    private void transferTo(InputStream in, OutputStream out) throws IOException {
        byte[] buffer = new byte[8192];
        int bytesRead;
        while ((bytesRead = in.read(buffer)) != -1) {
            out.write(buffer, 0, bytesRead);
        }
    }

    // 使用示例
    public static void main(String[] args) {
        AmrToMp3Converter converter = new AmrToMp3Converter();
        String url = "https://example.com/audio.amr";

        try (InputStream is = converter.getInputStream(url)) {
            // 处理输入流（例如播放、保存、转发等）
            // 这里仅演示读取并丢弃
            byte[] buffer = new byte[4096];
            while (is.read(buffer) != -1) {}
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

    关键说明

    1. 文件类型判断：仅通过 URL 路径后缀判断，若 URL 不含扩展名（如 REST 接口），可改用 URLConnection.getContentType() 或读取文件头（AMR 文件头为 #!AMR）。为简洁起见，代码采用后缀方式。

    2. 临时文件清理：

      • AMR 临时文件在转换后立即删除。

      • MP3 临时文件在返回的流关闭时自动删除（通过覆写 close()）。

      • 若流未被关闭，临时文件将残留；建议始终在 try-with-resources 中使用返回的流。

    3. 依赖说明：JAVE2 的 Encoder.encode() 可能抛出 EncoderException，需妥善处理。

    4. 性能：转换涉及磁盘读写，适合中小文件。若需流式实时转换，需采用管道 + FFmpeg 进程，复杂度较高。

    你遇到的 InputFormatException: Invalid argument 错误，通常意味着 JAVE2 在尝试读取或处理你的 AMR 文件时失败了。

    方案二：手动安装 FFmpeg 并配置系统路径

    如果方案一仍报错，可能是 JAVE2 自带的 FFmpeg 无法识别你的特定 AMR 文件。这时，让 JAVE2 使用你系统上安装的、更新版本的 FFmpeg 会更可靠 。

    1. 安装 FFmpeg

    你遇到的 InputFormatException: Invalid argument 错误，即使更换了明确的 3.5.0 依赖后依然存在，这说明问题可能不在依赖本身，而在于输入文件或JAVE2 内部调用的 FFmpeg 参数。我们需要一步步诊断真实原因。

    1. 启用 JAVE2 调试日志，获取详细错误信息

    JAVE2 内部会调用 FFmpeg 并捕获其输出，默认情况下这些输出是隐藏的。通过设置系统属性可以打印出实际执行的 FFmpeg 命令及其错误输出，这对定位问题至关重要。

    在调用转换代码之前（例如在 main 方法开头）添加：
    java

    System.setProperty("sound.out", "debug");
System.setProperty("ffmpeg.log.level", "debug");

    运行你的程序，控制台会输出类似这样的信息：
    text

    Executing: C:\Users\xxx\AppData\Local\Temp\jave\ffmpeg-amd64-3.5.0.exe -i C:\Users\xxx\AppData\Local\Temp\audio_123.amr ...

    并附带 FFmpeg 的原始错误输出。请仔细查看其中是否有诸如 Invalid data found when processing input 或 Unknown encoder 等关键信息。

    2. 手动验证 AMR 文件的有效性

    可能是下载的 AMR 文件本身损坏，或并非真正的 AMR 格式（比如文件名是 .amr 但内容是其他格式）。建议：

    • 打印临时文件路径，在转换前保留临时文件不删除，然后手动用 FFmpeg 测试。

    • 使用 FFmpeg 命令行（从 JAVE2 打印的路径找到 FFmpeg 可执行文件）直接执行转换，例如：
      text

      "C:\Users\xxx\AppData\Local\Temp\jave\ffmpeg-amd64-3.5.0.exe" -i "C:\Users\xxx\AppData\Local\Temp\audio_123.amr" -acodec libmp3lame -b:a 128k -ar 44100 -ac 2 "C:\Users\xxx\AppData\Local\Temp\output.mp3"

      观察是否成功，并查看错误信息。

    如果命令行也失败，说明 FFmpeg 无法识别该文件，可能是文件格式问题或缺少解码器。可以尝试用系统安装的完整版 FFmpeg 测试。

    3. 检查 FFmpeg 是否支持 AMR 解码

    有时 JAVE2 自带的 FFmpeg 可能因为编译选项没有包含 AMR 支持（虽然官方构建通常包含，但为了确认）。可以手动下载一个完整的 FFmpeg 并指定其路径：

    • 从 FFmpeg 官网 下载 Windows 完整版（例如 ffmpeg-release-full.7z），解压后将 bin 目录加入系统 PATH，或者在代码中通过系统属性指定路径：


    java

    System.setProperty("ffmpeg.home", "D:\\ffmpeg\\bin"); // 替换为你的 FFmpeg 目录

    然后再次运行转换，看是否成功。如果成功，说明 JAVE2 自带的 FFmpeg 版本有问题，你可以始终使用系统 FFmpeg。

    4. 备选方案：直接调用 FFmpeg 进程（绕过 JAVE2）

    如果以上步骤都无法解决问题，你可以直接使用 Java 的 ProcessBuilder 调用 FFmpeg，这样能完全控制命令和错误处理。以下是一个简单的工具类示例：
    java

    import java.io.*;
import java.nio.file.Files;

public class FfmpegAmrToMp3 {

    public static InputStream convert(InputStream amrStream) throws IOException, InterruptedException {
        // 创建临时文件
        File amrTemp = File.createTempFile("audio_", ".amr");
        File mp3Temp = File.createTempFile("audio_", ".mp3");

        // 保存输入流到临时文件
        try (FileOutputStream fos = new FileOutputStream(amrTemp)) {
            amrStream.transferTo(fos);
        }

        // 构建 FFmpeg 命令（假设 ffmpeg 在 PATH 中，否则写全路径）
        ProcessBuilder pb = new ProcessBuilder(
                "ffmpeg",
                "-i", amrTemp.getAbsolutePath(),
                "-acodec", "libmp3lame",
                "-b:a", "128k",
                "-ar", "44100",
                "-ac", "2",
                "-y",  // 覆盖输出文件
                mp3Temp.getAbsolutePath()
        );
        pb.redirectErrorStream(true); // 合并错误输出
        Process process = pb.start();

        // 读取进程输出（可选，可打印到日志）
        try (BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println("[FFmpeg] " + line);
            }
        }

        int exitCode = process.waitFor();
        if (exitCode != 0) {
            throw new IOException("FFmpeg 转换失败，退出码：" + exitCode);
        }

        // 删除 AMR 临时文件
        amrTemp.delete();

        // 返回 MP3 文件流，并设置自动删除
        return new FileInputStream(mp3Temp) {
            @Override
            public void close() throws IOException {
                super.close();
                mp3Temp.delete();
            }
        };
    }
}

    2. 执行两步转换 (SILK → PCM → MP3)
      利用解码器将 SILK 文件转换为中间 PCM 数据，然后再调用我们之前成功的 JAVE2 命令行方式（这次不用 -f silk），将 PCM 数据编码为 MP3。因为 PCM 是标准的裸数据格式，JAVE2 的内置 FFmpeg 可以完美处理 。

    以下是完整的参考代码实现：
    java

    import ws.schild.jave.*;
import java.io.*;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.StandardCopyOption;

public class WechatAudioConverter {

    /**
     * 使用内置silk解码器 + JAVE2内置FFmpeg进行转换
     * @param wechatAmrStream 原始微信语音文件的输入流
     * @return 转换后的 MP3 文件输入流
     */
    public static InputStream convertWechatAmrToMp3(InputStream wechatAmrStream)
            throws IOException, InterruptedException {

        // 1. 创建临时文件：纯SILK数据、中间PCM文件、最终MP3文件
        Path tempSilk = Files.createTempFile("pure_wechat_", ".silk");
        Path tempPcm = Files.createTempFile("audio_", ".pcm");
        Path tempMp3 = Files.createTempFile("converted_", ".mp3");

        // 2. 获取内置的silk-v3-decoder可执行文件
        Path silkDecoder = extractBuiltinDecoder();

        try {
            // 3. 跳过微信私有的第一个字节，写入纯SILK文件
            try (InputStream rawInput = wechatAmrStream) {
                long skipped = rawInput.skip(1);
                if (skipped != 1) {
                    throw new IOException("无法跳过微信私有头部（文件可能为空或已损坏）");
                }
                Files.copy(rawInput, tempSilk, StandardCopyOption.REPLACE_EXISTING);
            }

            // 4. 第一步：调用内置decoder，将SILK解码为PCM
            ProcessBuilder silkDecodePb = new ProcessBuilder(
                    silkDecoder.toString(),
                    tempSilk.toString(),
                    tempPcm.toString()
            );
            silkDecodePb.redirectErrorStream(true);
            Process silkProcess = silkDecodePb.start();
            // 打印解码器输出（可选）
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(silkProcess.getInputStream()))) {
                String line;
                while ((reader).ready() && (line = reader.readLine()) != null) {
                    System.out.println("[SilkDecoder] " + line);
                }
            }
            int silkExitCode = silkProcess.waitFor();
            if (silkExitCode != 0) {
                throw new IOException("SILK解码失败，退出码: " + silkExitCode);
            }

            // 5. 第二步：使用JAVE2内置FFmpeg，将PCM编码为MP3
            FFMPEGLocator locator = new DefaultFFMPEGLocator();
            String ffmpegPath = locator.getFFMPEGExecutablePath();

            // PCM转MP3命令
            // -f s16le: 指定PCM格式为16位小端
            // -ar 24000: 采样率与微信语音一致
            // -ac 1: 单声道
            ProcessBuilder ffmpegPb = new ProcessBuilder(
                    ffmpegPath,
                    "-y",
                    "-f", "s16le",
                    "-ar", "24000",
                    "-ac", "1",
                    "-i", tempPcm.toString(),
                    "-acodec", "libmp3lame",
                    "-ab", "32k",
                    tempMp3.toString()
            );
            ffmpegPb.redirectErrorStream(true);
            Process ffmpegProcess = ffmpegPb.start();
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(ffmpegProcess.getInputStream()))) {
                String line;
                while ((reader).ready() && (line = reader.readLine()) != null) {
                    System.out.println("[FFmpeg] " + line);
                }
            }
            int ffmpegExitCode = ffmpegProcess.waitFor();
            if (ffmpegExitCode != 0) {
                throw new IOException("FFmpeg PCM转MP3失败，退出码: " + ffmpegExitCode);
            }

            // 6. 返回MP3文件流
            return Files.newInputStream(tempMp3);

        } finally {
            // 清理临时文件
            Files.deleteIfExists(tempSilk);
            Files.deleteIfExists(tempPcm);
            // 注意：tempMp3 由调用者使用完毕后自行删除
        }
    }

    /** 从资源目录提取内置的silk-v3-decoder到临时目录 */
    private static Path extractBuiltinDecoder() throws IOException {
        String resourcePath = "/linux/decoder"; // 你的资源路径
        Path tempDir = Files.createTempDirectory("silk_decoder_");
        Path decoderPath = tempDir.resolve("decoder");

        try (InputStream is = WechatAudioConverter.class.getResourceAsStream(resourcePath)) {
            if (is == null) {
                throw new IOException("内置解码器资源未找到: " + resourcePath);
            }
            Files.copy(is, decoderPath, StandardCopyOption.REPLACE_EXISTING);
        }

        // 设置可执行权限 (Linux必需)
        if (!decoderPath.toFile().setExecutable(true)) {
            throw new IOException("无法设置解码器执行权限");
        }

        return decoderPath;
    }
}

import java.io.*;

import java.nio.file.Path;

 * 微信语音文件转换工具（.amr → .mp3）
 * 策略：跳过微信私有头部 → 使用内置 silk_decoder 解码为 PCM → 使用 JAVE2 内置 FFmpeg 编码为 MP3

    /**
     * 将微信语音输入流转换为 MP3 输入流
     * @param wechatAmrStream 原始微信语音的输入流（调用者负责关闭）
     * @return MP3 文件的输入流（调用者使用完毕后必须关闭）

        Path tempMp3 = Files.createTempFile("wechat_mp3_", ".mp3");

        try {

                    "-y",

    静态编译会将所有库（包括 glibc）直接打包进可执行文件，使其能在任何 Linux 上运行，但文件体积会稍大（约几百 KB）。

    在 Ubuntu 上执行静态编译

    1. 安装静态编译所需的库：
      bash

      sudo apt update
sudo apt install build-essential libc6-dev-static

    2. 进入 silk 目录，使用静态编译参数重新编译：
      bash

      cd ~/silk-v3-decoder/silk
make clean   # 清理之前编译的文件
gcc -static -O3 -Iinterface -Isrc -Itest -o decoder *.c

      这会将所有 .c 文件编译成一个完全静态的 decoder 可执行文件。

    3. 验证静态编译结果：
      bash

      file decoder
ldd decoder   # 应该输出 "not a dynamic executable" 或 "statically linked"

      如果 ldd 显示 statically linked，说明静态编译成功。

    4. 将新生成的 decoder 文件再次复制到 Windows 共享文件夹，并放入你的 Java 项目资源中替换旧文件。

    🛠️ 第一步：修改 converter.sh，支持自定义 FFmpeg 路径

# File: converter.sh (modified for custom ffmpeg)
# Date: August 19th, 2016

# Link: https://github.com/kn007/silk-v3-decoder

# Flag: not define   ----  not define, convert a file
#       other value  ----  format, convert a folder, batch conversion support
# Requirement: gcc ffmpeg (but now we use custom ffmpeg via FFMPEG_PATH)

# 支持自定义 FFmpeg 路径（通过环境变量 FFMPEG_PATH 传入）
FFMPEG="${FFMPEG_PATH:-ffmpeg}"

# Colors

if [ ! -r "$cur_dir/silk/decoder" ]; then
    echo -e "${WHITE}[Notice]${RESET} Silk v3 Decoder is not found, compile it."

        let CURRENT+=1
        $cur_dir/silk/decoder "$1/$line" "$2/$line.pcm" > /dev/null 2>&1

        fi

        ffmpeg_pid=$!

        rm "$2/$line.pcm"
        [ ! -f "$2/${line%.*}.$3" ] && echo -e "[$CURRENT/$TOTAL]${YELLOW}[Warning]${RESET} Convert $line false, maybe ffmpeg no format handler for $3." && continue

    exit
done

    ☕ 第三步：Java 代码集成

    修改你的工具类，将资源中的脚本和解码器解压到临时目录，并通过环境变量 FFMPEG_PATH 传递 JAVE 内置的 FFmpeg 路径。
    java

    import ws.schild.jave.*;
import java.io.*;
import java.nio.file.*;
import java.util.Comparator;

public class WechatAudioConverter {

    public static InputStream convertToMp3(InputStream wechatAmrStream) throws IOException, InterruptedException {
        // 1. 创建临时工作目录
        Path workDir = Files.createTempDirectory("wechat_conv_");
        Path inputFile = workDir.resolve("input.amr");
        Path outputFile = workDir.resolve("input.mp3"); // 脚本会生成此文件

        try {
            // 2. 保存微信语音流（跳过头部后的纯数据）到临时文件
            try (InputStream rawInput = wechatAmrStream) {
                rawInput.skip(1); // 跳过微信私有头
                Files.copy(rawInput, inputFile, StandardCopyOption.REPLACE_EXISTING);
            }

            // 3. 解压 converter.sh 和 silk/decoder
            extractScriptAndDecoder(workDir);

            // 4. 获取 JAVE 内置 FFmpeg 路径
            FFMPEGLocator locator = new DefaultFFMPEGLocator();
            String ffmpegPath = locator.getFFMPEGExecutablePath();

            // 5. 执行 converter.sh（单个文件模式）
            ProcessBuilder pb = new ProcessBuilder(
                    "bash", workDir.resolve("converter.sh").toString(),
                    inputFile.toString(),  // 输入文件
                    "mp3"                  // 输出格式
            );
            pb.directory(workDir.toFile());
            pb.environment().put("FFMPEG_PATH", ffmpegPath); // 关键：传递自定义 ffmpeg
            pb.redirectErrorStream(true);

            Process process = pb.start();
            // 打印日志（便于调试）
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    System.out.println("[converter] " + line);
                }
            }
            int exitCode = process.waitFor();
            if (exitCode != 0) {
                throw new IOException("转换失败，退出码: " + exitCode);
            }

            // 6. 返回生成的 MP3 文件流
            if (!Files.exists(outputFile)) {
                throw new IOException("输出文件未生成: " + outputFile);
            }
            return Files.newInputStream(outputFile);

        } finally {
            // 7. 清理临时目录（注意：返回的流使用完后才关闭，所以不能在这里删除 outputFile）
            // 可以注册 JVM 钩子，或由调用者负责删除。简单起见，这里不自动删除，留给调用者处理。
            // 如果希望立即删除，可以在调用方关闭流后调用 Files.delete(outputFile) 并删除 workDir。
        }
    }

    private static void extractScriptAndDecoder(Path workDir) throws IOException {
        // 解压 converter.sh
        Path scriptTarget = workDir.resolve("converter.sh");
        try (InputStream is = WechatAudioConverter.class.getResourceAsStream("/native/linux/converter.sh")) {
            if (is == null) throw new IOException("converter.sh not found in resources");
            Files.copy(is, scriptTarget, StandardCopyOption.REPLACE_EXISTING);
        }
        scriptTarget.toFile().setExecutable(true);

        // 创建 silk 子目录并解压 decoder
        Path silkDir = workDir.resolve("silk");
        Files.createDirectories(silkDir);
        Path decoderTarget = silkDir.resolve("decoder");
        try (InputStream is = WechatAudioConverter.class.getResourceAsStream("/native/linux/silk/decoder")) {
            if (is == null) throw new IOException("decoder not found in resources");
            Files.copy(is, decoderTarget, StandardCopyOption.REPLACE_EXISTING);
        }
        decoderTarget.toFile().setExecutable(true);
    }
}
