一 需求背景

背景

  1. 企微渠道活跃度提升过程中，需要通过部分服务性和福利性的内容来引导，以物流售后KP消息和券临期提醒为首次尝试。
  2. 人工建立触达任务的效果基本沉淀出来了，逐步交由系统进行发放，这样先做到提效的目的，再让不同渠道联动触达成为可能。

二、目标

2.1：业务目标

  1. 服务型消息接入一个月后，企微活跃门店数上升1w家。
  2. 5月份可以引入渠道联动的形式进行转化收割。

2.2：系统目标

  1. 建立系统触达任务对接服务、模型、以及统一的接入模式, 为以后全集团2c、2b业务场景业务事件引导提供全新随时触达模式.
  2. 沉淀任务处理批次模型, 以支持系统任务、人工任务. 大任务拆分、业务拆分场景下, 分批处理下并需要流程监控和统计的服务能力.
  3. 触达任务核心执行时间条件, 支持任意定制化, 为后期系统对接各场景业务提供更高的扩展性.


三、名词解释

任务模板：即任务创建框创建的含有任务业务名称(如系统任务名称)、业务触达逻辑、触达内容模板、发送圈店逻辑配置、发送时间等属性所属的任务定义模型.

任务实例：运行时模型, 业务任务、批量执行任务的调度单位, (如现金券任务, 每隔N天执行一次, 每次执行都为一个批次), 一个任务模板可关联, 一个或多个任务批次, 代表当前任务多次的批量调度单位(单批次下可含有全量或多).

任务明细：运行时模型, 触达任务执行的最小单位. 涵盖单个触达模型中的所有内容(触达对象、触达内容、发送方、任务明细执行状态).

任务策略：触达任务发送过程中的业务规则总和,  包含业务事件、固定规则、以及部分业务场景下业务扩展点.

业务事件：对应任务模板中业务触达逻辑具体在业务侧的触发事件(如门店物流异常) 主要指代业务侧的事件, 某种意义上是业务对接过程中和其他业务线开发边界梳理的核心.


此次包含的业务事件如下:

现金券临期3天
88VIP满减券临期3天
88VIP现金券临期3天
88VIP常规88元已发券月初提醒
88VIP超88元已发券月初提醒
P0物流月初提醒


固定规则：此处特指可通过DMP指标量化和对接的规则 .

业务扩展点：无法通过固定规则和业务事件涵盖的逻辑, 或作为业务上某些业务定制逻辑或case的补充(如针对dmp t+1数据实时部分逻辑的补充和过滤, 或者物流信息产品逻辑层面需要对单个触达对象单位时间内相同的触达消息做归并等)

CREATE TABLE `t_touch_task_instance` (
  `id` bigint(32) unsigned NOT NULL AUTO_INCREMENT COMMENT '自增主键',
  `creator` varchar(32) NOT NULL COMMENT '创建人',
  `editor` varchar(32) NOT NULL COMMENT '辑人',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `edit_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '编辑时间',
  `is_deleted` tinyint(1) NOT NULL DEFAULT '0' COMMENT '是否删除 (0.未删除 1.删除)',
  `biz_type` tinyint(4) NOT NULL COMMENT '业务场景 1、触达中心人工创建任务 2、触达中心系统任务',
  `biz_id` varchar(32) NOT NULL COMMENT '业务id, 触达中心场景为task_id',
  `instance_level` tinyint(4) DEFAULT NULL COMMENT '实例等级 1、紧急；2、一般；3、不紧急',
  `instance_type` tinyint(4) DEFAULT NULL COMMENT '实例内容类型 1活动营销类 2单品推广类 3奶粉行情类 4人设养成类 5官方通知类 6主推项目类',
  `instance_status` tinyint(4) NOT NULL COMMENT '实例状态 1、待发送，2、正在发送，3、发送完成，4、发送暂停，5、发送取消，6、二次推送',
  `use_ab_test` tinyint(4) NOT NULL DEFAULT '0' COMMENT '是否开启AB测试 0不开启 1开启',
  `channel_type` tinyint(4) NOT NULL DEFAULT '1' COMMENT '发送渠道 1涂色通道 2企微官方通道',
  `start_time` datetime DEFAULT NULL COMMENT '开始时间',
  `end_time` datetime DEFAULT NULL COMMENT '结束时间',
  PRIMARY KEY (`id`),
  KEY `idx_biz_type_biz_id` (`biz_type`,`biz_id`)
) ENGINE=InnoDB AUTO_INCREMENT=100000 DEFAULT CHARSET=utf8mb4 COMMENT='触达任务实例';
